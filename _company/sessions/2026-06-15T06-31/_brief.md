# 📋 작업 브리프

**원 명령:** [프로젝트: 인스타그램 기반 청정 정치 밸런스 플랫폼 '시소(Seesaw)']

우리는 인스타그램 3배열 그리드(좌:진보, 중:긴코랭킹, 우:보수)에 맞춰 매일 팩트 기반의 정치 콘텐츠를 자동으로 생산하는 파이프라인을 구축하고자 한다. 내 PC 환경 및 GitHub(main 브랜치)와 연동하여 다음 요구사항을 수행하는 백엔드 자동화(Python) 코드를 구현하고 세팅해 줘.

1. 실시간 데이터 수집 및 진영 분류
- 네이버 뉴스 RSS/주요 언론사 API 및 유튜브 Data API를 활용해 최근 24시간 동안의 정치 이슈 및 인물 데이터를 실시간 크롤링할 것.
- 수집된 콘텐츠를 분석하여 '진보 진영(좌)'과 '보수 진영(우)' 언론 및 유튜버들이 다루는 핵심 이슈로 상성별 자동 분류할 것.

2. '어제의 TOP 1' 및 '긴코 랭킹(🤥)' 팩트 정제
- 진영별로 하루 동안 언급량이 가장 급증한 핵심 인물을 각각 1명씩 선정할 것 (어제의 TOP 1).
- 공인의 말 바꾸기를 추적하기 위해, 특정 인물의 과거 발언(뉴스 기사, 국회 속기록 데이터 등)과 현재의 반대되는 행보/발언 데이터 세트를 찾아 매칭할 것.
- 주관적인 비방이나 판단은 일절 배제하고, 100% 사실(Fact)과 출처 링크만 나열하여 명예훼손 리스크를 원천 차단할 것.

3. 우측 Antigravity 연동용 데이터 구조화 (JSON 저장)
- 정제된 데이터를 우측 Antigravity 에이전트가 읽어서 카드뉴스 이미지로 변환할 수 있도록 매일 'seesaw_today.json' 파일로 자동 저장할 것.
- JSON 데이터 구조 스펙:
  * [LEFT]: 진보 핵심 이슈 요약, 주요 뉴스/유튜브 리스트, 포인트 컬러 코드(Cyan)
  * [CENTER]: 긴코 랭킹 인물명, 과거 발언(날짜/출처/내용), 현재 행보(날짜/출처/내용), 포인트 컬러 코드(Purple / 개혁신당 색상 중복 회피)
  * [RIGHT]: 보수 핵심 이슈 요약, 주요 뉴스/유튜브 리스트, 포인트 컬러 코드(Red)

4. 24시간 자동 가동 설정
- 이 전체 프로세스가 매일 새벽 지정된 시간(예: 아침 06:00)에 자동으로 실행되어 JSON 데이터를 갱신하도록 크론탭(Crontab) 또는 내부 스케줄러 환경을 세팅할 것.

지시한 스펙에 맞게 크롤러 및 데이터 분류 스크립트 작성을 시작하고, 완료되면 GitHub 동기화 및 실행 대기 상태를 보고해라.

## 요약
인스타그램 기반 정치 밸런스 플랫폼 '시소(Seesaw)'를 위한 정치 이슈 자동 수집 및 JSON 저장 백엔드 파이프라인 구축을 요청합니다.
