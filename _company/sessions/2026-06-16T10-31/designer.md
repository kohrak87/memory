### 📊 잠재 수익 분포 시각화 프로토타입: 최종 디자인 스펙

**목표:** 데이터 신뢰도($\lambda$) 변화에 따른 잠재 수익($KPI_{final}$)의 분포를 시각적으로 극대화하여, 데이터 기반 의사결정을 지원하는 KPI 대시보드 구현.

#### 1. 브랜드 스타일 및 색상 (Color & Typography)

이 디자인은 '오류 없는 답변'이라는 회사 정체성에 맞춰 **신뢰성**과 **명확성**을 최우선으로 합니다.

*   **메인 컬러 (Primary - 신뢰/성장):** `#007BFF` (Blue, 명확성과 전문성을 상징)
*   **보조 컬러 (Secondary - 위험/기회):** `#28A745` (Green, 긍정적 수익 및 성장 방향)
*   **경고/위험 컬러 (Alert - 신뢰도 하락):** `#DC3545` (Red, 데이터 신뢰도($\lambda$)가 낮을 때 위험 경고)
*   **배경 컬러 (Background):** `#F8F9FA` (Light Gray, 시각적 피로도를 줄이고 데이터에 집중)
*   **텍스트/기본 폰트:**
    *   **Font Family:** Inter (가독성과 현대적인 느낌 강조)
    *   **Heading Font:** Inter Bold (H1, H2 레벨에 사용)
    *   **Body Font:** Inter Regular (데이터 라벨 및 설명에 사용)

#### 2. 레이아웃 구조 (Layout Structure: Dashboard Layout)

전체 대시보드는 `DashboardLayout` 컴포넌트를 기반으로 하며, 데이터 흐름 우선순위(신뢰성 > 효율성 > 자동화)를 반영하여 구성합니다.

*   **구조:** 사이드바 + 탑바 + 메인 영역 통합 (`DashboardLayout`)
*   **사이드바 (Navigation):** 좌측에 고정. 주요 섹션으로 연결 (예: Overview, Data Reliability, Profit Distribution, Action Items).
*   **탑바 (Header):** 상단에 핵심 KPI 요약 및 시스템 상태 표시.
*   **메인 영역 (Main Area):** 시각화 차트와 상세 데이터가 배치되는 공간.

#### 3. 핵심 컴포넌트 사양 (Core Component Specifications)

##### A. 데이터 신뢰도($\lambda$) 중심의 분포 시각화 (핵심 차트)

*   **차트 유형:** **Scatter Plot (산점도)** + **Heatmap Overlay (히트맵 오버레이)**
*   **목적:** 잠재 수익 분포를 $\lambda$ 값에 따라 표현.
*   **X축 (가로):** 데이터 신뢰도($\lambda$) 값 ($0.0 \sim 1.0$)
*   **Y축 (세로):** 잠재 수익 ($KPI_{final}$) 값 (최소값 ~ 최대값)
*   **데이터 포인트 표현:**
    *   각 데이터 포인트는 $\lambda$ 축 상에서 위치하며, **색상 강도(Intensity)**를 통해 분포의 밀도를 나타냅니다.
    *   **컬러 매핑 규칙:**
        *   $\lambda \ge 0.9$: 진한 녹색 (안정적/고신뢰)
        *   $0.7 \le \lambda < 0.9$: 중간 녹색 (주의 필요)
        *   $\lambda < 0.7$: 주황색/빨간색 (위험 영역, 신뢰도 낮음)

##### B. KPI 대시보드 요약 섹션 (KPI Summary Cards)

각 수익화 모델별로 별도의 카드(Card Component)를 사용하여 핵심 정보를 요약합니다.

*   **카드 제목:** 수익화 모델 이름 (예: Model A, Model B-B)
*   **핵심 지표 (Primary Metric):** $KPI_{final}$ 값 (가장 큰 폰트로 표시)
*   **신뢰도 지표 (Reliability Indicator):** $\lambda$ 값 (보조 컬러로 표시하며, $\lambda < 0.8$일 경우 경고 아이콘 표시)
*   **추가 정보:** $Margin_{Weighted}$ 및 $T_{wait}$ 요약치.

##### C. 데이터 신뢰도($\lambda$) 변화 시뮬레이션 영역 (Interactive Element)

사용자가 $\lambda$ 값을 슬라이더로 조작할 수 있는 인터랙티브 영역을 정의합니다.

*   **슬라이더:** $\lambda$ 값 ($\text{Min}=0.5$ 에서 $\text{Max}=1.0$)
*   **결과 시각화:** 슬라이더 이동에 따라, Scatter Plot 상의 데이터 포인트 색상 분포와 $KPI_{final}$의 평균/분산 변화를 실시간으로 반영하여 보여줍니다. (이것이 프로토타입의 핵심 가치입니다.)

#### 4. 최종 Figma 구현 지침 요약 (Actionable Blueprint)

1.  **레이아웃:** `DashboardLayout`을 기본 틀로 하며, 좌측에 $\lambda$ 변화 시뮬레이션 컨트롤 패널을 배치합니다.
2.  **시각화:** 핵심은 Scatter Plot이며, 색상 코드는 신뢰도($\lambda$)를 직접적으로 반영해야 합니다.
3.  **인터랙션:** 사용자가 $\lambda$ 슬라이더를 움직이면, 차트 내의 모든 데이터 포인트와 KPI 요약 카드가 동기화되어 실시간으로 변화하도록 설정합니다.
4.  **컴포넌트:** KPI 카드(`KPI_Card`)는 `Primary Color`로 핵심 지표($KPI_{final}$)를 강조하고, $\lambda$ 값은 경고 색상으로 보조 표시합니다.
