# 🎨 Designer — 확정된 피드백 채널 구조(수집, 전달 프로세스)를 바탕으로 실제 고객 피드백 수집 폼 설계를 완료하고, 시스템 무결성 및 시각화 일관성을 유지하는 디자인을 적용하라.

🎨 Designer: 작업 시작합니다.

CEO의 지시에 따라, 확정된 피드백 채널 구조를 기반으로 고객 피드백 수집 폼을 설계하고, 기존 시스템 무결성 및 시각화 일관성을 유지하는 디자인을 적용하겠습니다. 이는 Launch Runbook의 최종 검증 단계에서 필수적인 사용자 경험(UX) 확보 작업입니다.

먼저, 이전 세션에서 확정된 **브랜드 비주얼 시스템**과 **피드백 채널 구조**를 참조하여 폼 디자인 스펙을 도출합니다.

### 1. 피드백 수집 폼 디자인 스펙 (Design Specification)

이 설계는 기존에 확립된 브랜드 아이덴티티(컬러, 타이포그래피)와 데이터 흐름 우선순위(신뢰성 > 효율성)를 시각적으로 반영하여, 사용자가 신뢰감을 느끼면서도 쉽게 피드백을 제공하도록 설계합니다.

**A. 컬러 및 타이포그래피 적용**
*   **주요 색상 (Primary Color):** `#007BFF` (신뢰성과 기술적 정밀함을 상징)
*   **보조 색상 (Secondary Color - 경고/중요):** `#FFC107` (데이터 신뢰도($\lambda$)의 중요성을 강조)
*   **배경:** `#F8F9FA` (깔끔하고 오류 없는 답변의 톤 유지)
*   **폰트 스타일:** 산세리프 계열. 제목은 Bold, 본문은 Regular로 설정하여 가독성과 시스템 무결성을 시각적으로 표현합니다.

**B. 레이아웃 및 구조 (Layout & Structure)**
폼은 '신뢰성'과 '효율성'이라는 두 가지 핵심 축을 중심으로 구성됩니다.

| 영역 | 내용 | 디자인 목표 | 적용 규칙 |
| :--- | :--- | :--- | :--- |
| **Header** | 폼 제목 및 안내 문구 | 명확한 목적 제시 | Primary Color (`#007BFF`) 사용, 간결한 타이포그래피 |
| **Section 1: 신뢰도 확인 (Trust Check)** | 시스템 무결성 관련 질문 (예: "제공된 데이터의 정확성에 대해 어떻게 평가하십니까?") | 시스템 일관성 강조 | $\lambda$ 점수 또는 체크박스 기반 구조, 보조 색상(`#FFC107`)으로 중요도 표시 |
| **Section 2: 피드백 입력 (Feedback Input)** | 핵심 피드백 내용 입력 영역 (자유 서술형) | 효율적인 정보 수집 | 충분한 여백 확보, 명확한 입력 필드 라벨링 |
| **Section 3: 전달 프로세스 확인 (Delivery Process Check)** | 피드백이 어떻게 처리될지 안내 (수집 $\rightarrow$ 검토 $\rightarrow$ 전달) | 투명성 제공 | 단계별 진행 바(Progress Bar) 또는 흐름도 시각화 |
| **Footer** | 제출 버튼 및 개인 정보 동의 | 명확한 Call to Action (CTA) | Primary Color (`#007BFF`)로 강조된 최종 CTA 버튼 |

**C. 시스템 무결성 반영 요소**
*   모든 입력 필드 옆에는 해당 데이터가 시스템에 미치는 영향(예: "이 응답은 $\lambda$ 점수 계산에 영향을 줍니다")을 작게 표시하여, 사용자가 입력하는 행위 자체가 시스템의 일부임을 인지하게 합니다.

### 2. 최종 디자인 산출물 요약 (Visual Summary)

실제 Figma 구현을 위해 아래와 같이 핵심 디자인 변수를 정리합니다.

*   **🎨 Color Palette:** `{Primary: #007BFF, Secondary: #FFC107, Background: #F8F9FA}`
*   **✒️ Typography Style:** `Sans-serif (모든 텍스트는 명료성 우선)`
*   **📐 Layout Principle:** `Trust > Input > Process` 흐름을 따르는 수직적 위계 구조.

이 스펙은 **Launch Runbook**에 포함될 최종 디자인 가이드의 일부로 사용되어야 합니다.
