# 📐 사용자 피드백 수집 폼: 최종 UI/UX 디자인 스펙 확정

## 1. 핵심 디자인 원칙 (Brand Alignment)

**브랜드 톤 반영:** '오류 없는 답변'이라는 브랜드 정체성에 맞추어, 모든 데이터(기술 상태 포함)는 불확실성을 최소화하고 명확한 신뢰도를 제공해야 합니다.

*   **신뢰도 시각화 목표:** 사용자에게 시스템이 현재 어떤 상태인지 **즉각적이고 오류 없이** 전달하여 심리적 안정감을 제공합니다.
*   **레이아웃 전략:** 정보의 계층 구조를 명확히 하여, 사용자가 피드백을 제공하는 것 외에 '시스템 상태'를 인지하도록 유도합니다.

## 2. 프로토타입 상세 스펙 (Synchronization & Feedback Integration)

기존의 프로토타입(`sessions/2026-06-16T20-01/designer.md`, `sessions/2026-06-16T20-16/designer.md`)에서 도출된 요소를 통합하여 최종 구조를 확정합니다.

### 2.1. Sync_Status_Bar 디자인 정의 (기술 안정성 시각화)

이 요소는 사용자에게 시스템의 현재 상태(Sync Verification 결과)를 명확히 전달하는 핵심 지표입니다.

| 상태 (State) | 시각적 표현 (Visual Cue) | 색상 코드 (Color Code) | 설명 (UX Meaning) |
| :--- | :--- | :--- | :--- |
| **Pending Sync** | 회전 중 애니메이션 (`Spinner`) | `#FFA500` (주황색/경고) | 데이터 동기화 진행 중, 잠시 기다려 주세요. |
| **Synced (High Trust)** | 체크 마크와 녹색 배경 | `#4CAF50` (녹색/성공) | 모든 데이터가 안정적으로 동기화됨. 시스템 신뢰도 높음. |
| **Sync Error** | 느낌표와 빨간색 경고 표시 | `#F44336` (빨간색/오류) | 동기화 실패 또는 데이터 불일치 발생. 즉각적인 확인 필요. |

### 2.2. 피드백 수집 폼 레이아웃 구조 (Data Flow Integration)

피드백 입력 영역과 시스템 상태 표시 영역을 명확히 분리하여 사용자가 혼란 없이 정보를 제공하도록 설계합니다.

1.  **Header & Status Bar (상단):**
    *   `Sync_Status_Bar`를 화면 상단에 고정 배치하여, 사용자가 피드백을 시작하기 전에 시스템의 현재 안정성 상태를 인지하게 합니다. (가장 중요한 정보 우선)
2.  **Feedback Section (중앙):**
    *   사용자 피드백 항목(텍스트 입력 필드, 선택 옵션 등)을 명확하고 간결하게 제시합니다.
    *   입력 필드 옆에는 **필수 입력 여부(`*`)**와 함께 관련 데이터 흐름의 중요도($\lambda$)를 미세하게 표시하여, 사용자가 어떤 정보가 중요한지 인지하도록 돕습니다.
3.  **Action & Submission (하단):**
    *   '제출(Submit)' 버튼은 명확한 CTA 색상(`Primary Color`)을 사용하며, 제출 시 발생하는 데이터 처리 과정을 짧게 안내합니다.

## 3. 최종 디자인 가이드라인 요약

*   **컬러 팔레트:** 브랜드 컬러를 기본으로 하되, 시스템 상태 표시를 위해 **경고(빨강), 진행(주황), 성공(녹색)**의 보조 색상을 엄격하게 사용합니다.
*   **타이포그래피:** 정보의 위계를 명확히 하기 위해 산세리프 계열을 유지하되, 기술적 수치($\lambda$, $KPI$)는 명확하고 읽기 쉬운 폰트 크기와 굵기로 강조합니다.
*   **인터랙션:** `Sync_Status_Bar`의 애니메이션은 최소화하여 시스템의 안정성을 침해하지 않도록 하되, 상태 변화는 즉각적으로 반영되어야 합니다.
