이 작업은 기존에 확정된 브랜드 컬러/타이포 시스템과 데이터 흐름 우선순위(신뢰성 > 효율성)를 반영하여, 기술적 안정성과 사용자 경험의 동기화가 시각적으로 명확하게 드러나는 프로토타입을 목표로 합니다.

### 🚀 1단계: 디자인 스펙 및 프로토타입 구조 확정 (Figma Blueprint)

시스템 무결성(`lint_test` 결과)과 UX 일치도 확보를 위해, 고객 피드백 폼은 **'신뢰성 강조형(Trust-Focused)'** 레이아웃을 채택하며, 데이터 흐름의 중요도를 시각적으로 반영합니다.

#### 1. 디자인 시스템 적용 (Brand System Application)

기존에 확정된 브랜드 스타일을 프로토타입에 즉시 적용합니다.

*   **컬러 팔레트 적용:** 핵심 컬러(`Primary: #007AFF`, `Secondary: #333333`, `Background: #F9F9F9`)를 사용하여 신뢰감과 명료함을 강조합니다.
*   **타이포그래피 적용:** 본문 텍스트는 가독성을 위해 Sans-serif 계열(예: Pretendard 또는 Inter)을 사용하며, 데이터 필드는 명확한 구분을 위해 Weight 변화를 활용합니다.

#### 2. 고객 피드백 수집 폼 레이아웃 (Prototype Structure)

**목표:** 사용자가 정보를 입력하는 과정에서 **데이터의 신뢰성($\lambda$)**이 지속적으로 강조되어야 합니다.

| 영역 | 구성 요소 | 디자인 지침 및 목적 | 적용 스타일 |
| :--- | :--- | :--- | :--- |
| **A. 헤더 (Trust Bar)** | 제목, 데이터 신뢰도 표시 | "데이터 기반 피드백 수집" 명시. 시스템 무결성(Stability)이 이 프로세스의 전제임을 암시. | `Primary Color` 배경, 볼드 타이포 |
| **B. 핵심 KPI 요약** | 현재까지의 데이터 흐름 요약 (선택 사항) | 사용자가 어떤 데이터를 제공하고 있는지 직관적으로 인지하게 하여 입력 동기화 유도. | 카드형 레이아웃, `Secondary Color` 테두리 |
| **C. 피드백 입력 섹션** | 질문 및 답변 필드 | 각 질문은 명확하게 분리하며, 필수 항목(`*`)을 강조하여 데이터 누락 방지. | 명확한 여백(Padding), Input Field는 얇은 테두리 적용. |
| **D. 동기화 피드백 (Sync Feedback)** | 시스템 상태 표시줄 | "현재 데이터 흐름 안정성: ✅ 통과됨" 또는 "실시간 동기화 중..." 등 실시간 시스템 상태를 미니멀하게 표시하여 UX/기술 일치도를 시각적으로 증명. | `Success Color` 또는 `Warning Color` 사용 |
| **E. CTA (Call to Action)** | 제출 버튼 | 명확하고 강력한 행동 유도 문구. | `Primary Color` 배경, 높은 대비(Contrast) |

### 🛠️ 2단계: 프로토타입 시각화 (Visual Execution)

위의 스펙을 바탕으로 Figma 내에서 다음 디자인 요소를 즉시 제작합니다.

1.  **Figma 컴포넌트 생성:**
    *   `Form_Input_Field` (Label + Input)
    *   `KPI_Summary_Card` (데이터 시각화 블록)
    *   `Sync_Status_Bar` (시스템 동기화 상태 표시줄)

2.  **레이아웃 배치:**
    *   사용자 흐름에 따라 A → B → C → D → E의 순서로 섹션을 구성합니다. 특히 **D. 동기화 피드백**은 사용자가 데이터를 입력하는 전체 과정에서 항상 상단 또는 측면에 노출되어 시스템 신뢰도를 끊임없이 확인하게 만듭니다.

3.  **최종 검토:**
    *   제작된 프로토타입이 기존의 브랜드 컬러와 타이포그래피 가이드라인을 100% 준수하는지 확인합니다. (Self-Check: $\lambda$ 점수의 중요도를 시각적으로 극대화했는가?)
